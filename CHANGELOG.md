- 收口菜单编辑弹窗的变更记录探测异常；同步后端不可用时不再影响编辑流程。

## [1.0.2] - 2026-04-14

### 🐛 修复
- 修复生产环境未注入 `NEXT_PUBLIC_API_BASE_URL` 时，前端错误请求 `/sync/*`、`/spaces/*`、`/auth/*` 路径导致同步与加入空间接口返回 404 的问题。

- 改进同步 404 错误提示，直接提示检查 `NEXT_PUBLIC_API_BASE_URL` 并在修改后重新执行构建。

### ✅ 验证
- 新增和扩展 Vitest 覆盖默认 `/api` 与自定义 API base 两种场景。
- 当前单元测试已通过，生产构建已通过。

## [1.0.0] - 2025-04-13

### 🎉 初始版本发布

> 从今天起，不再纠结"今天吃什么"。

### ✨ 功能
- **随机抽选**：支持"自己做"、"叫外卖"和自定义组合模板抽选。
- **菜单管理**：添加 / 编辑 / 删除菜品，支持标签分类、材料清单、烹饪步骤和图片上传。
- **标签体系**：自定义标签（菜系、类别、自定义），支持多标签筛选与搜索。
- **组合模板**：按规则批量抽取，如"1 荤 + 1 素 + 1 汤"。
- **历史记录**：自动记录每次抽选结果，随时回看"今天吃了什么"。
- **忌口管理**：标记忌口菜品，抽选时自动规避，支持一键查看与取消。
- **多设备同步**：通过邀请码加入同一共享空间，多台设备实时同步菜单数据，匿名即可使用。
- **PWA 支持**：可安装到手机桌面，离线可用，已针对移动端进行交互和布局优化。

### 🛠️ 技术
- 前端：Next.js 14 (App Router) + React 18 + TypeScript + Tailwind CSS
- 后端：Next.js API Routes + better-sqlite3（单文件 SQLite）
- 客户端存储：Dexie.js (IndexedDB)
- 测试：Vitest + jsdom，47 项单元测试全部通过

### 🐛 修复
- 修复种子数据缺失 `spaceId` 导致的编辑后无法同步问题。
- 修复移动端弹窗按钮和标签区域可能溢出的问题。
- 修复菜单页 React 异步状态更新警告。
