# 项目功能文档

> 基于当前仓库代码状态整理，更新时间：2026-04-21。  
> 用途：在后续新增功能、改字段、扩同步链路前，先快速了解项目现状与改动入口。

## 1. 项目定位

这是一个围绕“今天吃什么”决策场景构建的 Web 应用，当前已经不是纯演示版，而是一个具备完整主流程的可用产品：

- 支持本地使用：菜单、标签、模板、历史、设置都可离线工作
- 支持共享空间：可创建/加入空间，多设备共享菜单数据
- 支持账号体系：邮箱 + 密码登录，忘记密码 / 首次设密走 QQ SMTP
- 支持随机决策：单抽、组合抽、去重、忌口、想吃加权、个人权重
- 支持互动：点赞、评论、匿名评论
- 支持备份：本地私有数据 ZIP 导入导出，旧 JSON 兼容导入
- 支持安装：PWA 可装到手机桌面

## 2. 当前架构

### 2.1 技术栈

- 前端：Next.js 14 App Router + React 18 + TypeScript + Tailwind CSS
- 客户端本地存储：Dexie / IndexedDB
- 服务端：Next.js Route Handlers
- 服务端数据库：better-sqlite3 + SQLite
- 测试：Vitest

### 2.2 数据层分工

当前项目是“双层存储”架构：

1. 未加入共享空间时：
   - 主要数据保存在浏览器 IndexedDB
   - 所有页面直接读写 Dexie

2. 已加入共享空间时：
   - 前端仍然先写 IndexedDB
   - 记录会被标记为 `pending`
   - `lib/http-sync-engine.ts` 负责把本地变更推送到 `app/api/*`
   - 服务端写入 SQLite
   - 再把远端数据拉回本地完成对齐

3. 账号与密码：
   - 服务端 `users / sessions / password_reset_tokens` 负责账号、会话和重置令牌
   - 登录方式为邮箱 + 密码
   - QQ SMTP 只承担忘记密码 / 首次设密邮件，不参与日常登录

### 2.3 关键数据流

- UI 页面主要使用 `useLiveQuery + db` 直接读本地库
- 业务写操作主要经过 `lib/space-ops.ts`
- 同步入口在 `lib/http-sync-engine.ts`
- 服务端表定义和触发器在 `lib/db-server.ts`
- 通用表同步转换逻辑在 `lib/sync-api.ts`

## 4. 核心业务规则

### 4.1 菜单项模型

菜单项分为两类：

- `recipe`：菜谱
- `takeout`：外卖

公共字段在 `lib/types.ts` 的 `MenuItem` 中定义。当前已包含：

- 基础字段：名称、标签、创建/更新时间
- 菜谱字段：图片、材料、步骤、心得
- 外卖字段：店铺、地址
- 同步字段：`spaceId`、`profileId`、`syncStatus`、`version`

### 4.2 随机抽选规则

随机逻辑位于 `lib/roll.ts`，当前规则如下：

- 支持按类型筛选：菜谱 / 外卖 / 全部
- 支持按标签筛选
- 支持默认去重天数
- 支持全局关闭去重或单次忽略去重
- 忌口项会直接被排除
- “想吃”会放大权重，当前倍率为 `3`
- 个人权重参与加权随机
- 组合模板抽取时，同一轮结果内不重复
- 若近期去重后无候选项，会自动放宽限制并标记 `ignoredDedup`

### 4.3 历史与版本

- 抽取历史保存在本地 `rollHistory`
- 共享数据的变更记录保存在服务端 `change_logs`
- `menu_items` 支持从历史快照恢复旧版本
- 变更记录由 SQLite trigger 自动生成，不依赖前端显式写日志

### 4.4 同步策略

当前同步是“本地优先 + SSE 事件通知 + 增量拉取 + 轮询回退”的混合模式：

- 本地先写 Dexie
- 共享数据写入时标记为 `pending`
- 删除操作额外记录到 `pendingDeletions`
- 点赞使用确定性主键，避免多端重复
- 评论按 `updatedAt` 做最后写入生效
- 标签同步时支持“同名同类型标签映射合并”
- 首次进入共享空间时做全量拉取
- 后续通过 `GET /api/sync/events` 建立 SSE 事件流
- 收到事件后通过 `GET /api/sync/delta` 拉取增量变更并推进本地 `cursor`
- 当 SSE 不可用时回退到指数退避轮询
- 本地冲突会进入 `syncConflicts`，可在设置页决议“接受远端”或“保留本地版本”
- 所有 `/api/sync/*` 和变更记录读取接口都要求：
  - 当前已登录
  - 当前账号属于目标 `spaceId`
  - 服务端写入时只认当前会话对应的成员身份，不再信任客户端上传的 `profileId / spaceId`

### 4.5 同步数据矩阵

| 数据 | 范围 | 写入入口 | 同步触发 | 删除/清空同步 | 另一设备恢复路径 |
| --- | --- | --- | --- | --- | --- |
| `settings.defaultDedupDays` / `dedupEnabled` / `theme` | 账号私有 | 设置页、主题切换 | `profile-state` dirty key | 远端快照权威，缺失时回到默认值 | 登录同账号并进入同空间后拉取 `/api/sync/profile-state` |
| `avoidances` | 账号私有 | 忌口页、详情页、菜单删除清理 | `profile-state` dirty key | 取消忌口或删除菜单会记录对应 key | 同账号同 profile 快照恢复 |
| `wishes` | 账号私有 | 想吃切换、菜单删除清理 | `profile-state` dirty key | 取消想吃或删除菜单会记录对应 key | 同账号同 profile 快照恢复 |
| `favorites` | 账号私有 | 收藏切换、菜单删除清理 | `profile-state` dirty key | 取消收藏或删除菜单会记录对应 key | 同账号同 profile 快照恢复 |
| `personalWeights` | 账号私有 | 权重设置、权重删除、菜单删除清理 | `profile-state` dirty key | 删除权重或删除菜单会记录对应 key | 同账号同 profile 快照恢复 |
| `menuGroups` | 账号私有 | 场景清单创建、重命名、删除 | `profile-state` dirty key | 删除清单会删除清单和条目 | 同账号同 profile 快照恢复 |
| `menuGroupItems` | 账号私有 | 加入清单、移出清单、排序、菜单删除清理 | `profile-state` dirty key | 移出、删除清单、删除菜单会记录对应 key | 同账号同 profile 快照恢复 |
| `rollHistory` | 账号私有 | 单抽、组合抽、多人抽选、历史清空 | `profile-state` dirty key / reset | 清空历史使用集合 reset，不会被旧设备复活 | 同账号同 profile 快照恢复，最多保留 500 条 |
| `menuItems` | 空间共享 | 菜单新增、编辑、删除、图片 URL 更新 | shared sync pending + SSE/delta | 删除写入 `pendingDeletions` 并通过 change log 传播 | 同空间成员全量拉取或增量拉取 |
| `tags` | 空间共享 | 标签新增、编辑、删除 | shared sync pending + SSE/delta | 删除写入 `pendingDeletions` | 同空间成员全量拉取或增量拉取 |
| `comboTemplates` | 空间共享 | 模板新增、编辑、删除 | shared sync pending + SSE/delta | 删除写入 `pendingDeletions` | 同空间成员全量拉取或增量拉取 |
| `likes` | 空间共享互动 | 点赞、取消点赞 | shared sync pending + SSE/delta | 取消点赞写入 `pendingDeletions`，稳定主键去重 | 同空间成员全量拉取或增量拉取 |
| `comments` | 空间共享互动 | 评论新增、编辑、删除、撤回 | shared sync pending + SSE/delta | 删除写入 `pendingDeletions`，撤回重新写入 | 同空间成员全量拉取或增量拉取 |
| `profiles` / `space` | 空间共享元数据 | 创建空间、加入空间、绑定旧身份 | 服务端 session + SQLite | 不通过客户端删除 | 登录后 session 恢复并写入本地当前空间指针 |
| `pendingDeletions` / `syncConflicts` / cursor / connection meta | 设备本地同步状态 | 同步引擎内部 | 不作为账号数据同步 | 由同步结果清理 | 不跨设备恢复 |
| `lastBackupAt` / 当前身份缓存 / 备份文件 | 设备本地 | 设置页、本地存储、用户下载 | 不同步 | 仅本机有效 | 不跨设备恢复 |

当前 `profile-state` 使用“具体 dirty key”合并：本机只会覆盖自己实际改过的 key；未改过的本地旧数据以远端快照为准，避免旧设备把已删除的忌口、收藏、清单或历史重新同步回来。

### 4.6 多人抽选与私有偏好

- 多人抽选和多人推荐只由服务端读取参与成员的私有偏好。
- 前端只提交参与成员 ID，不接收其他成员的原始忌口、想吃、权重、收藏或分组。
- 聚合规则为：任意成员忌口命中则排除；缺失个人权重按 `1` 处理；权重按参与人数求平均；想吃参与加分；收藏不影响随机。
- 接口只返回抽选结果、推荐列表和汇总理由，例如“成员平均偏好较高”或“2/3 成员想吃”。

### 4.7 图片处理

- 当前菜谱图片会先在前端压缩，再通过 `/api/images/menu-item/[menuItemId]` 上传到本地文件存储
- `MenuItem.imageUrl` 现在保存的是服务器可访问的相对 URL，而不是 base64
- 服务端和客户端都包含对 legacy base64 图片的迁移逻辑
- 这样可以显著降低 SQLite 与同步 payload 的体积

### 4.8 备份与重置边界

- 个人备份当前定义为**本地私有数据备份**
- 导出内容包括：`settings`、`avoidances`、`personalWeights`、`rollHistory`、本地 `menuItems / tags / comboTemplates` 以及本地图片文件
- UI 默认导出 ZIP 备份包：`backup.json + images/`
- 共享空间同步来的菜单、标签、模板、点赞、评论不会进入个人备份
- 本地模式下“重置数据”会清空本地数据并恢复示例数据
- 共享空间下“重置数据”会保留当前身份，只清空当前设备上的本地缓存，然后重新同步当前空间
- 当前仍在共享空间时，不允许直接导入个人备份，避免把本地备份和空间数据混在一起
- 旧 JSON 备份仍兼容导入
- SMTP 凭据只允许放在 `.env.local` 或部署平台环境变量中，不进入 Git

## 7. 当前已知边界与遗留点

这些点不影响核心使用，但在继续开发前应心里有数。

### 7.1 共享同步仍是单机 SSE，不是多实例广播

共享空间已经从“单纯轮询”升级为 `SSE + 增量拉取 + 轮询回退`。  
它已经比早期版本更接近实时协作，但仍然不是数据库级实时推送，也没有做到多实例广播。

### 7.2 成员维度功能已起步，但还没真正展开

虽然已有：

- `profiles` 表
- `fetchProfiles()` 方法
- 设置页成员列表
- 菜单详情里的创建者展示

但权限、操作归属筛选、在线状态、更细粒度的成员协作视角仍未展开。

### 7.3 图片链路已脱离 base64，但备份与离线缓存还可以继续优化

当前图片已经改为本地文件存储，主链路性能压力显著下降；后续仍可继续关注：

- 更大图片集下的磁盘占用与缓存策略
- ZIP 备份在大量图片场景下的导出耗时
- PWA 离线缓存对图片命中率的进一步优化

## 8. 收口结论

这份旧文档到此为止已经完成收口。  
本轮已经完成的关键收口项包括：

- 账号体系从邮箱验证码切换为邮箱 + 密码

- `/api/sync/*` 与变更记录读取统一加上登录态和空间成员校验
- 旧匿名用户只允许绑定当前设备本地保存的旧身份
- 未绑定旧成员的真实 `profileId` 不再通过共享响应对外暴露
- 共享图片与空间详情接口已经补齐权限校验
- 文档、README、CHANGELOG 与当前代码行为已对齐

后续如果要继续扩功能，例如多空间切换、多实例实时分发、更深的成员协作视角、更复杂的推荐与偏好协同，应进入新的需求文档，不再追加到本文。
