# 数据迁移方案：本地版 → 网络版

> 本文档说明用户如何从当前的纯本地 PWA 版本迁移数据到未来的网络版（如有）。

## 1. 导出本地数据

1. 打开应用，进入「设置」页面。
2. 点击「导出备份（JSON）」。
3. 浏览器会自动下载一个 JSON 文件，妥善保存。

## 2. 导入到网络版

1. 在网络版应用中登录账号。
2. 进入「设置」→「导入备份」。
3. 选择之前导出的 JSON 文件。
4. 系统会自动完成以下映射：
   - 旧数据的 `id` 会保留为本地引用。
   - 服务端会为每条记录分配新的 `remoteId`。
   - `syncStatus` 会被标记为 `synced`。
   - 所有数据会关联到当前登录的 `userId`。

## 3. 字段映射说明

| 本地字段 | 网络版行为 |
|---------|-----------|
| `MenuItem.id` | 保留，同时写入 `remoteId` |
| `syncStatus` | 导入后统一设置为 `synced` |
| `userId` | 导入后设置为当前登录用户 ID |
| `rollHistory` | 完整保留，支持时间线回溯 |
| `comboTemplates` | 完整保留，含自定义模板 |
| `settings` | 部分保留（去重天数等），备份提醒不再必要 |

## 4. 兼容性保障

- JSON Schema 版本号（`schemaVersion`）用于校验：版本不兼容时会给出明确提示。
- 缺失字段会自动填充默认值（如 `syncStatus = "local"`）。
- Dexie 数据库版本升级机制（`version(2)`）会自动为旧数据库补全新字段。

## 5. 注意事项

- 本地版和网络版的数据是相互独立的；导入是**单向复制**，不会双向同步。
- 如果网络版已有数据，导入时建议选择「覆盖」或「合并」策略（以网络版实际交互为准）。
- 建议定期导出本地 JSON 备份，以防浏览器清理缓存导致数据丢失。
